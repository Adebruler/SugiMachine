Background

The SugiMachine is a recreational practice in machine learning to predict Pokémon's types based on their images (with the added bonus of assigning to types any picture you can imagine).

The model is named after renowned manga illustrator Ken Sugimori, who is most famous for his official illustrations of Pokémon.

Using Pokémon for data analysis practice was a natural choice because their a large but discrete number of Pokémon (809) that are diverse but catergorical and have many stats of their own.

Process

The model was trained using a Keras CNN on Ken Sugimori's Pokémon illustrations which were scraped from Bulbapedia using Beautiful Soup.

Complications

Every Pokémon has either one or two types, and it is not reasonable to assume which from a picture alone. The model was given all types during training, but keras is natively optimized to return one prediction. Sugimachine returns the top two predictions (if a second is given any confidence at all,) irrespective of the input.

A reasonable modified accuracy metric returns 0.5 for each type correct on a two-type Pokémon, and returns a 1.0 if the first-guess was correct for a one-type Pokémon or 0.5 if the second-guess was correct. Otherwise 0.0 accuracy is returned. By this metric, the model is 38% accurate.

Because there are a relatively large number of types (18) for a relatively small number of Pokémon (809)--and because the Pokémon are diverse within their types--the model is prone to overtraining, which may be seen on the Model Progress chart to the left.

Future Updates
